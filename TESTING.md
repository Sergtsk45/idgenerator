# Инструкция по проверке генерации PDF АОСР

## ✅ Быстрая проверка (тестовый скрипт)

Самый простой способ - запустить тестовый скрипт:

```bash
npx tsx test-pdf-generation.ts
```

Скрипт создаст два PDF файла в корне проекта:
- `test-output-with-materials.pdf` - с таблицей материалов
- `test-output-no-materials.pdf` - без материалов (с текстом "Согласно проектной документации")

**Что проверить в созданных PDF:**
1. ✅ Все плейсхолдеры заменены на реальные данные
2. ✅ Даты в формате "24 января 2026" (не ISO)
3. ✅ В первом PDF есть таблица с 2 строками материалов
4. ✅ Во втором PDF есть строка "Согласно проектной документации"
5. ✅ Русский текст отображается корректно (шрифт Roboto)

## 🔧 Проверка через API (если есть акт в БД)

### Шаг 1: Запустите сервер

```bash
npm run dev
```

Сервер должен запуститься на `http://localhost:5000` (или другом порту из `PORT`).

### Шаг 2: Создайте акт через API (если его нет)

```bash
# Создать акт с шаблонами
curl -X POST http://localhost:5000/api/acts/create-with-templates \
  -H "Content-Type: application/json" \
  -d '{
    "dateStart": "2026-01-20",
    "dateEnd": "2026-01-24",
    "location": "г. Москва, ул. Тестовая, д. 1",
    "templateIds": ["foundation-prep"]
  }'
```

Ответ вернёт `id` акта (например, `{"id": 1, ...}`).

### Шаг 3: Экспортируйте акт в PDF

```bash
# Замените :id на реальный ID акта
curl -X POST http://localhost:5000/api/acts/1/export \
  -H "Content-Type: application/json" \
  -d '{
    "templateIds": ["foundation-prep"],
    "formData": {
      "actNumber": "123",
      "city": "Москва",
      "objectName": "Тестовый объект"
    }
  }'
```

Ответ вернёт список созданных файлов:
```json
{
  "success": true,
  "files": [
    {
      "templateId": "foundation-prep",
      "filename": "AOSR_AOSR-01_1_1234567890.pdf",
      "url": "/api/pdfs/AOSR_AOSR-01_1_1234567890.pdf"
    }
  ]
}
```

### Шаг 4: Откройте PDF в браузере

Перейдите по URL из ответа:
```
http://localhost:5000/api/pdfs/AOSR_AOSR-01_1_1234567890.pdf
```

## 🖥️ Проверка через UI (полный сценарий)

### Шаг 1: Запустите сервер

```bash
npm run dev
```

### Шаг 2: Откройте приложение в браузере

Обычно это `http://localhost:5000` (или порт из `PORT`).

### Шаг 3: Перейдите на страницу "Акты"

В нижней навигации выберите "Акты" (или откройте `/acts`).

### Шаг 4: Создайте новый акт

1. Нажмите кнопку создания акта (обычно "+" или "Создать акт")
2. Выберите даты начала и окончания работ
3. Выберите один или несколько шаблонов (например, "Подготовка основания")
4. Нажмите "Создать" или "Экспортировать"

### Шаг 5: Проверьте результат

- PDF должен открыться в новой вкладке браузера
- Проверьте, что все данные подставлены корректно
- Проверьте формат дат и таблицу материалов

## 🐛 Отладка проблем

### Проблема: PDF не генерируется

**Проверьте:**
1. Есть ли файл `server/templates/aosr/aosr-template.json`?
2. Есть ли шрифты в `server/fonts/`?
3. Есть ли ошибки в консоли сервера?

**Логи:**
```bash
# Смотрите логи сервера при генерации
# Ошибки будут в консоли, где запущен `npm run dev`
```

### Проблема: Плейсхолдеры не заменяются

**Проверьте:**
1. Откройте `server/templates/aosr/aosr-template.json` - должны быть плейсхолдеры вида `{{actNumber}}`
2. Проверьте, что в `server/pdfGenerator.ts` функция `replacePlaceholdersDeep` вызывается
3. Проверьте логи сервера на предупреждения `[AOSR] Missing placeholder value`

### Проблема: Таблица материалов пустая или неправильная

**Проверьте:**
1. В шаблоне должна быть таблица с заголовком "Наименование"
2. Функция `findMaterialsTableNode` должна найти эту таблицу
3. Проверьте логи на ошибку `[AOSR] Materials table not found`
