# Техзадание: Tablet UI для TelegramJurnalRabot

**Дата**: 2026-03-10  
**Статус**: Draft  
**Назначение**: Документация для расширения мобильного приложения (mobile-first) на планшетные экраны и режимы работы.

---

## Назначение этой папки

Папка содержит полное техническое описание задачи адаптации существующего **mobile-first Telegram MiniApp** (TelegramJournalRabot) для работы на планшетах и больших экранах. Проект ориентирован на **строительную логистику** (журнал работ, ведомости объёмов, акты АОСР) и требует не просто responsive CSS, а архитектурных изменений в shell, навигации и управлении состоянием.

**Целевая аудитория**: frontend-разработчик, имеющий опыт с React + Tailwind + TypeScript.

## Карта экранов приложения

Приложение состоит из 5 основных экранов в нижней навигации (**BottomNav**):

### 1. **ВОР** (`/works`)
- **Назначение**: Импорт и просмотр ведомостей объёмов работ из Excel
- **Компоненты**: таблица с фиксированными заголовками, поиск, импорт XLSX
- **Tablet требования**: горизонтальная таблица с несколькими видимыми колонками, sidebar для фильтров
- **Документация**: см. `01-foundation-platform-shell.md` → **Таблицы**

### 2. **График работ** (`/schedule`)
- **Назначение**: Диаграмма Ганта, редактирование задач, разделение задач (split), управление материалами
- **Компоненты**: временная шкала, карточки задач, диалоги редактирования
- **Tablet требования**: горизонтальная диаграмма Ганта на полный экран, боковая панель деталей
- **Документация**: см. `01-foundation-platform-shell.md` → **Модальные диалоги и Sheet**

### 3. **Акты АОСР** (`/acts`)
- **Назначение**: Генерация актов освидетельствования скрытых работ (PDF)
- **Компоненты**: список актов, calendar для выбора периода, выбор шаблонов
- **Tablet требования**: два столбца (список слева, детали справа)
- **Документация**: см. `01-foundation-platform-shell.md` → **Двухколоночная компоновка**

### 4. **Журнал работ** (`/worklog`)
- **Назначение**: Таблицы с обработанными записями из чата
- **Компоненты**: табы, горизонтальная прокрутка табов, таблицы
- **Tablet требования**: полноширинные таблицы с горизонтальным скроллом, sticky-заголовки
- **Документация**: см. `01-foundation-platform-shell.md` → **Таблицы**

### 5. **Исходные данные** (`/source-data`)
- **Назначение**: Дашборд исходных данных по объекту (материалы, документы, ответственные лица)
- **Компоненты**: sticky текущий объект, горизонтальные карточки, блоки редактирования
- **Tablet требования**: grid-компоновка вместо горизонтального scroll
- **Документация**: см. `01-foundation-platform-shell.md` → **Компоновка grid**

### 6. **Главная (чат)** (`/`)
- **Назначение**: Журнал работ в формате чата, голосовой ввод, AI-распознавание
- **Компоненты**: список пузырьков сообщений, форма ввода, микрофон
- **Tablet требования**: двухколоночный макет (список слева, детали справа) или полноэкранный чат
- **Документация**: см. `00-development-plan.md` → **Приоритеты экранов**

### 7. **Мои объекты** (`/objects`)
- **Назначение**: Список строительных объектов с фильтрацией
- **Tablet требования**: grid-компоновка вместо карточек в столбик
- **Документация**: см. `01-foundation-platform-shell.md` → **Компоновка grid**

### 8. **Настройки** (`/settings`)
- **Назначение**: Язык интерфейса, аккаунт, параметры
- **Tablet требования**: стандартная компоновка (изменений минимально)

## Ключевые выводы из анализа кода

1. **Shell сейчас mobile-first**:
   - Максимальная ширина контейнеров: `max-w-md` (448px)
   - Header: `sticky top-0`, fixed high (`h-14`)
   - BottomNav: `fixed bottom-0`, fixed high (`h-16` + `pb-safe`)
   - Основной контент: `pb-24` (отступ под BottomNav)

2. **Telegram MiniApp интеграция**:
   - WebApp SDK подключен в `index.html`
   - CSS-переменные темы Telegram (--tg-theme-*)
   - Хуки: `useTelegram()`, `useTelegramMainButton()`, `useTelegramBackButton()`
   - Safe-area поддержка (`pb-safe`, `pt-safe`)
   - Fullscreen режим через `WebApp.expand()`

3. **State Management**:
   - TanStack Query для серверного состояния
   - Zustand для локального состояния (язык)
   - Object switching: текущий объект хранится серверно (`users.current_object_id`)
   - Query cache invalidation нужна при изменении объекта

4. **Компоненты shadcn/ui**:
   - 40+ компонентов (Dialog, Sheet, Drawer, Table, Tabs, Accordion и т.д.)
   - Tailwind-based, полностью кастомизируемые
   - Accessible (Radix UI под капотом)

5. **Текущие ограничения**:
   - Нет responsive навигации (меню гамбургера вместо sidebar)
   - Таблицы с горизонтальным скроллом на мобильных
   - Диалоги/Sheet часто fullscreen (нужны modal dialogs для tablet)
   - Landscape режим не оптимизирован
