# Руководство по импорту материалов из Excel

## Обзор

Функционал массового импорта материалов позволяет администраторам загружать справочник материалов из Excel-файла в глобальный каталог системы.

## Формат Excel-файла

### Структура колонок

| Колонка | Название | Тип | Обязательное | Описание |
|---------|----------|-----|--------------|----------|
| A | № | number | Нет | Порядковый номер (игнорируется системой) |
| B | Наименование | string | **Да** | Название материала (до 500 символов) |
| C | Ед. изм. | string | Нет | Единица измерения (м³, т, шт и т.д.) |
| D | ГОСТ/ТУ | string | Нет | Стандарт или технические условия |
| E | Категория | string | Нет | material/equipment/product |

### Пример файла

```
№  | Наименование              | Ед. изм. | ГОСТ/ТУ         | Категория
1  | Бетон М300                | м³       | ГОСТ 26633-2015 | material
2  | Арматура А500С            | т        | ГОСТ 34028-2016 | material
3  | Кирпич керамический       | шт       | ГОСТ 530-2012   | material
4  | Экскаватор ЭО-4225А       | маш-час  |                 | equipment
5  | Плита перекрытия ПК 60-15 | шт       | ГОСТ 9561-2016  | product
```

### Требования к файлу

- **Формат**: .xlsx или .xls
- **Размер**: не более 10 МБ
- **Первая строка**: может содержать заголовки (автоматически пропускается)
- **Обязательное поле**: только "Наименование"
- **Кодировка**: любая (Excel автоматически обрабатывает)

## Использование

### Шаг 1: Подготовка файла

1. Создайте Excel-файл с колонками A-E
2. Заполните данные, начиная со второй строки
3. Убедитесь, что колонка "Наименование" заполнена для всех материалов
4. Сохраните файл в формате .xlsx

### Шаг 2: Импорт через веб-интерфейс

1. Откройте админ-панель → "Справочник материалов"
2. Нажмите кнопку "Импорт" в правом верхнем углу
3. Выберите подготовленный Excel-файл
4. Дождитесь завершения импорта
5. Проверьте результаты в toast-уведомлении

### Режимы импорта

#### Merge (по умолчанию)
- Создаёт новые материалы
- Обновляет существующие по совпадению названия (case-insensitive)
- Безопасный режим для добавления данных

#### Replace (будущая функция)
- Удаляет все существующие материалы (soft delete)
- Создаёт новые из файла
- Используется для полной замены справочника

## Обработка данных

### Валидация

**На клиенте (перед отправкой):**
- Проверка формата файла (.xlsx/.xls)
- Проверка размера файла (< 10 МБ)
- Парсинг структуры Excel
- Проверка наличия обязательных полей

**На сервере:**
- Валидация через Zod-схему
- Проверка длины названия (≤ 500 символов)
- Проверка категории (material/equipment/product)
- Транзакционная обработка

### Дубликаты

- Сравнение по названию (case-insensitive)
- "Бетон М300" = "бетон м300" = "БЕТОН М300"
- В режиме merge: последняя запись обновляет существующую
- Дубликаты в файле: последняя запись побеждает

### Пропущенные строки

Строки пропускаются если:
- Отсутствует название материала
- Название длиннее 500 символов
- Строка является заголовком
- Строка пустая или содержит менее 2 колонок

## Результаты импорта

После завершения импорта отображается статистика:

```
Импорт завершён
Получено: 150
Создано: 120
Обновлено: 25
Пропущено: 5
```

- **Получено**: количество строк в файле
- **Создано**: новые материалы, добавленные в каталог
- **Обновлено**: существующие материалы, данные которых изменены
- **Пропущено**: строки, не прошедшие валидацию

## Обработка ошибок

### Типичные ошибки

**"Размер файла превышает 10 МБ"**
- Решение: уменьшите количество строк или удалите лишние листы

**"Неподдерживаемый формат файла"**
- Решение: сохраните файл в формате .xlsx или .xls

**"В файле не найдено подходящих строк"**
- Решение: проверьте, что колонка B (Наименование) заполнена

**"Не удалось прочитать Excel файл"**
- Решение: файл повреждён, пересоздайте его

**"Ошибка при импорте материалов"**
- Решение: проверьте логи сервера, возможна проблема с БД

## Технические детали

### API Endpoint

```
POST /api/admin/materials-catalog/import
Content-Type: application/json
X-Admin-Override: true (только в dev-режиме)

Body:
{
  "mode": "merge",
  "items": [
    {
      "name": "Бетон М300",
      "unit": "м³",
      "gostTu": "ГОСТ 26633-2015",
      "category": "material"
    }
  ]
}

Response:
{
  "received": 1,
  "created": 1,
  "updated": 0,
  "skipped": 0
}
```

### Права доступа

- Требуется роль **администратора**
- В dev-режиме: автоматический обход через `X-Admin-Override: true`
- В production: проверка через middleware `requireAdmin`

### Транзакции

Весь импорт выполняется в одной транзакции БД:
- Если возникает ошибка → rollback всех изменений
- Атомарность: всё или ничего
- Консистентность данных гарантирована

## Лучшие практики

1. **Перед импортом**: создайте резервную копию БД
2. **Тестирование**: сначала импортируйте небольшой файл (5-10 строк)
3. **Именование**: используйте понятные названия материалов
4. **Единицы измерения**: придерживайтесь стандартных обозначений (м³, т, м², шт)
5. **ГОСТ/ТУ**: указывайте актуальные стандарты
6. **Категории**: используйте правильные значения (material/equipment/product)

## Ограничения

- Максимальный размер файла: 10 МБ
- Максимальная длина названия: 500 символов
- Формат файла: только .xlsx и .xls
- Только администраторы могут импортировать
- Нет предпросмотра перед импортом (планируется в будущем)
- Нет истории импортов (планируется в будущем)

## Примеры использования

### Первичное заполнение справочника

1. Подготовьте полный список материалов в Excel
2. Импортируйте в режиме merge
3. Проверьте результаты в интерфейсе

### Обновление данных

1. Экспортируйте текущий справочник (вручную или через БД)
2. Внесите изменения в Excel
3. Импортируйте в режиме merge
4. Только изменённые материалы будут обновлены

### Миграция из другой системы

1. Экспортируйте данные из старой системы в Excel
2. Приведите к требуемому формату (5 колонок)
3. Импортируйте в режиме replace (будущая функция)
4. Проверьте полноту данных
