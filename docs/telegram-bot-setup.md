# Настройка Telegram-бота для MiniApp

Это руководство описывает процесс создания и настройки Telegram-бота для работы с вашим MiniApp.

## Предварительные требования

- Аккаунт в Telegram
- Доступ к @BotFather (официальный бот для создания ботов)

## Шаг 1: Создание бота через @BotFather

1. Откройте Telegram и найдите бота [@BotFather](https://t.me/botfather)
2. Отправьте команду `/newbot`
3. BotFather попросит вас ввести имя бота (отображаемое имя):
   ```
   Например: Журнал Работ
   ```
4. Затем введите username бота (должен заканчиваться на `bot`):
   ```
   Например: jurnal_rabot_bot
   ```
5. После успешного создания BotFather отправит вам **Bot Token** — сохраните его:
   ```
   Пример токена: 1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
   ```

## Шаг 2: Настройка Web App URL

После создания бота необходимо настроить URL вашего MiniApp:

### Вариант 1: Menu Button (рекомендуется)

1. Отправьте команду `/setmenubutton` в чате с @BotFather
2. Выберите вашего бота из списка
3. Нажмите "Edit menu button URL"
4. Введите URL вашего приложения:
   ```
   Для разработки: https://your-dev-url.com
   Для продакшена: https://your-production-url.com
   ```
5. Введите текст для кнопки меню (например: "Открыть журнал")

### Вариант 2: Inline Button

Вы также можете создать inline-кнопку в сообщениях бота:

```javascript
// Пример кода для отправки сообщения с кнопкой (Node.js)
bot.sendMessage(chatId, 'Откройте приложение:', {
  reply_markup: {
    inline_keyboard: [[
      {
        text: 'Открыть Журнал Работ',
        web_app: { url: 'https://your-app-url.com' }
      }
    ]]
  }
});
```

## Шаг 3: Настройка переменных окружения

1. Скопируйте Bot Token, полученный от BotFather
2. Добавьте его в файл `.env` вашего проекта:

```bash
# Telegram Bot Configuration
# Получите токен через @BotFather (команда /newbot)
# Формат: 1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
TELEGRAM_BOT_TOKEN=your_bot_token_here
```

3. Убедитесь, что файл `.env` добавлен в `.gitignore` (для безопасности)

## Шаг 4: Дополнительные настройки бота (опционально)

### Установка описания бота

```
/setdescription - Установить описание бота (отображается в профиле)
```

Пример описания:
```
Журнал работ для строительства. Ведение АОСР, управление графиком работ, контроль материалов.
```

### Установка короткого описания

```
/setabouttext - Короткое описание (отображается в списке ботов)
```

Пример:
```
Автоматизация строительной документации
```

### Установка изображения профиля

```
/setuserpic - Загрузить аватар бота
```

### Настройка команд

```
/setcommands - Настроить список команд бота
```

Пример команд:
```
start - Запустить приложение
help - Помощь
settings - Настройки
```

## Шаг 5: Тестирование

1. Найдите вашего бота в Telegram по username
2. Нажмите `/start`
3. Нажмите на кнопку меню (три горизонтальные линии рядом с полем ввода)
4. Должно открыться ваше MiniApp

## Проверка интеграции

После настройки бота и запуска приложения проверьте:

1. **Инициализация WebApp**: Приложение должно автоматически развернуться на весь экран
2. **Данные пользователя**: В консоли браузера должны отображаться данные `Telegram.WebApp.initDataUnsafe`
3. **Тема**: Приложение должно адаптироваться под тему Telegram (светлая/темная)

## Безопасность

⚠️ **ВАЖНО:**

- **Никогда не публикуйте Bot Token** в публичных репозиториях
- Храните токен только в `.env` файле
- Добавьте `.env` в `.gitignore`
- На сервере используйте переменные окружения или секреты
- Всегда проверяйте подпись `initData` на сервере перед обработкой запросов

## Отладка

### Проблема: Бот не открывает приложение

**Решение:**
- Проверьте правильность URL в настройках menu button
- Убедитесь, что URL доступен по HTTPS (для продакшена)
- Для локальной разработки используйте ngrok или аналогичные сервисы

### Проблема: Не получаю данные пользователя

**Решение:**
- Проверьте, что скрипт `telegram-web-app.js` загружен до инициализации приложения
- Убедитесь, что вызван `Telegram.WebApp.ready()`
- Проверьте консоль браузера на наличие ошибок

### Проблема: Приложение не разворачивается на весь экран

**Решение:**
- Убедитесь, что вызван `Telegram.WebApp.expand()`
- Проверьте, что вызов происходит после `ready()`
